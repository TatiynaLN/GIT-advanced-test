git switch и git checkout

Переключение ветки:
git checkout "branch"
git switch "branch"


git checkout -b "branch"
git switch -с "new-branch"
git switch --create "new-branch"
git branch "new-branch"
примечание: git branch создает ветку, не переходя в нее

Cоздание новой ветки от текущего коммита и указание слежения на ветку удаленного репозитория:

git switch -c <branch> --track <remote>/<branch>
git checkout -b <branch> --track <remote>/<branch>

Создание ветки без родительского коммита

git switch --orphan <new_branch>




При совместной работе в одном репозитории нередко возникает необходимость получить ветку, в которой работает другой человек:

Создадим в удаленном репозитории ветку develop и зафиксируем в нем изменения.

Для этого можно клонировать проект в другую папку на компьютере:

$ git clone ссылка для клонирования репозитория
$ cd имя репозитория $ git checkout -b develop $ echo проект > README.md $ git add . $ git commit -m "добавлен README.md" $ git push origin develop



Получим последние изменения из удаленного репозитория:
$ git fetch origin
git checkout название ветки

Теперь рекомендуется при переключении веток писать так

git switch develop
или так, чтобы вернуться в предыдущую используемую ветку:
git switch -
