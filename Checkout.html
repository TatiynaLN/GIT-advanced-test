Checkout дополнительные возможности

Спецификация:

git checkout [-q] [-f] [-m] [<branch>]
Где,-q --quiet режим тишины, не выводить сообщения


git checkout [-q] [-f] [-m] --detach [<branch>]
где - -f --force при переключении веток позволяет продолжить работу, даже если индекс или рабочее дерево отличается от HEAD. Отменяет локальные изменения при переключении ветки.


git checkout [-q] [-f] [-m] [--detach] <commit>
-m --merge проводит слияние между текущей веткой, изменениями в рабочей директории и создает новую ветку

git checkout [-q] [-f] [-m] [[-b|-B|--orphan] <new_branch>] [<start_point>]
--detach переводит в состояние detached HEAD
--orphan создает новую ветку, первый коммит которой не будет иметь родительских коммитов, то есть будет самым первый коммитом в истории


Например, если использовать команду
git checkout --orphan clean_branch
