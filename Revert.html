Revert

Спецификация:

git revert [--[no-]edit] [-n] [-s] <commit> git revert (--continue | --skip | --abort | --quit)

Где:

  • --edit | -e позволяет редактировать название коммита перед коммитом
  • --no-edit не открывает радактор с вводом наименования коммита
  • --no-commit | -n не коммитить изменения автоматически

При конфликте когда нельзя автоматически провести revert, нужно разрешить проблемы и выполнить git revert --continue Чтобы отменить процесс используем git revert --abort

Если изменения были запушены в удаленный репозиторий, то reset использовать нельзя. Для таких случаев можно использовать команду revert на коммит.

Практическое использование:

  1. Вернем историю из коммитов содержащих 1.txt и 2.txt $ git reset --hard ff92d7b16f71245cbb4bc5682ad11ec24e934657$ git push
  2. Откатим ошибочные коммиты, выполнив $ git revert ff92d7b
  3. Система предложит наименование коммита, обычно оставляем как есть.

$ git log --oneline 85aff8b (HEAD -> master) Revert "2.txt" ff92d7b 2.txt 91de994 1.txt efa44b2 (origin/master) init simple project $ cat 2.txt cat: 2.txt: No such file or directory

  1. Запушим $ git push
  1. Таким образом мы сохранили историю и удалили изменения, которые произошли в коммите ff92d7b 2.txt
Дополнительная информация: https://git-scm.com/docs/git-revert
